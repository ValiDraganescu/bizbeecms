## Key decisions made this session
- **Country scope is a SET, not a single value.** Migration `0001` dropped the single `country` column on `users`/`invites` and added join tables `user_countries` / `invite_countries`. **No rows = global (all countries)** — an empty selection is valid and means all-countries. `sites.country` was deliberately LEFT as a single column (Site CRUD not built yet — decide its treatment, see Open questions).
- **Invite authz (server-enforced, `lib/invite/authz.ts`):** SuperAdmin + global Admins grant any country set or global; a country-scoped Admin grants only a NON-EMPTY SUBSET of their own countries (no global, no outside country); SuperAdmin never grantable via invite.
- **Email delivery:** Cloudflare Email Sending (`env.EMAIL.send()`, `send_email` binding) is the chosen path, but it's Beta/Workers-Paid and unavailable here (no CF auth, can't deploy). `lib/mail/send-invite.ts` is a seam that degrades to log + show the accept link in-app (`delivered:false`). This is why the user "got no email" — by design. `send_email` is a commented placeholder in `wrangler.jsonc`.
- **Security:** invite-link origin must come from a trusted `APP_ORIGIN` var, NOT request Host/X-Forwarded-Proto headers (Host Header Injection). Production throws if `APP_ORIGIN` unset; dev falls back to request host. Fixed in `ea92ebf` after an automated review flagged it.
- **i18n stays cookie-based** (not `/[locale]` path routing) — required for the Workers/OpenNext deploy gate. Don't reopen.
- **Build/dev workflow rule** (now in `.claude/prompt-extension.md`): NEVER run `next build` / `opennextjs-cloudflare build` while `npm run dev` is live — it corrupts `.next` and 500s the server. Stop dev → build → restart. Caused two self-inflicted 500s this session.

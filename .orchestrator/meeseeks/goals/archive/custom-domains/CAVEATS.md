# Caveats â€” custom-domains
Read every line before working. Each entry was learned the hard way by a previous Meeseeks.

- Project is Cloudflare-only. Custom hostnames must attach to the `bizbeecms.com` zone via Cloudflare APIs / wrangler â€” no external DNS/proxy infra.
- The SSO own-zone check already accepts any `*.bizbeecms.com` host, so `<slug>.site.bizbeecms.com` needs NO allowlist code change for SSO. Do not "fix" what isn't broken. Verify SSO end-to-end after each domain change rather than editing the allowlist speculatively.
- Don't loosen the SSO host allowlist. The 4-label + account-suffix anchoring (commits 01e7341, 9b1898d) exists to stop attacker-account lookalikes. Any change here is a security regression risk â€” verify it still rejects bad hosts.
- `npx opennextjs-cloudflare build` is the deploy gate; NEVER run it while `npm run dev` (port 3601) is running â€” it corrupts `.next` and 500s the dev server. Stop dev first.
- Per PM memory: REST only, no server actions (they 500 on Workers). Workers PBKDF2 capped at 100k iterations.
- Non-Pages Worker custom domain = a `routes:[{pattern,custom_domain:true}]` entry in its `wrangler.jsonc`; Cloudflare attaches the hostname+cert on `wrangler deploy`. workers.dev subdomain stays reachable unless explicitly disabled â€” keep it as a fallback.
- `cmsWorkerUrl()` (worker-url.ts) NO LONGER derives the account subdomain from APP_ORIGIN â€” it builds the workers.dev URL from `WORKERS_DEV_SUFFIX` (hosts.ts). So APP_ORIGIN is now free to be a custom domain. Don't "restore" the old APP_ORIGIN parsing. workers.dev is now the PERMANENT per-site CMS URL (the `.site.*` scheme was removed â€” see the SCHEME REMOVED caveat below).
- **SCHEME REMOVED 2026-06-19 (USER DECISION): the `*.site.bizbeecms.com` per-site scheme is GONE â€” do NOT reintroduce it.** The user ruled out the paid Advanced Certificate Manager `*.site.bizbeecms.com` wildcard cert, so sites stay PERMANENTLY on `bizbeecms-cms-<slug>.workers.dev`. Deleted this run: `SITE_HOST_SUFFIX`/`siteUrlForSlug()` (hosts.ts), the `*.site.bizbeecms.com/*` route (router/wrangler.jsonc), the leftmost-label slug-derivation branch (router/src/index.ts â€” now HOST_MAP-only for customer hostnames), the worker-url.ts "switch back to siteUrlForSlug" comment, and the `.site.*`-acceptance SSO test case. - **DO NOT switch sites to a bare `*.bizbeecms.com` router route. PROVEN BROKEN 2026-06-19:** a `*.bizbeecms.com/*` router route is too greedy â€” it SHADOWS our own one-level infra custom domains. This is WHY a two-level `.site.*` namespace was attempted, and why (with `.site.*` now removed) the only safe per-site addressing left is workers.dev (direct) + HOST_MAP customer hostnames (via router).

- PM "Open CMS"/site links now point at the per-site workers.dev URL (verified live: `bizbeecms-cms-test-1.workers.dev` â†’ 200). `cmsWorkerUrl` no longer gates on `CMS_WORKER_PREFIX` â€” it builds the URL for any non-empty worker name.

**CRITICAL — READ CAVEATS.md FIRST.** Two new traps hit this run, both now documented:
1. Collection MCP tools use INCONSISTENT `collection` arg naming: `create_collection` wants the bare label (`"offers"`), but `add_collection_field`/`add_collection_item`/`update_collection_item`/`query_collection` want the full `content_offers` name. Passing the wrong one either way gives `"collection not found"`.
2. Collection ITEM field values (`add_collection_item`/`update_collection_item`'s `values`) are single-locale plain strings — do NOT pass a `{en,fi,...}` locale object like you would for a translatable component PROP. It silently stores the string `"[object Object]"` with no error. Always plain strings for collection data; locale objects are only for component `props`/`propsSchema.default` on TRANSLATABLE (`(t)`) props.

Also: `add_collection_field`'s image-type field is `"asset"`, not `"image"` (that's the component-prop vocabulary, a different enum).

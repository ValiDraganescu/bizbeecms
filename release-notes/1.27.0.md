# CMS v1.27.0

_2026-07-22 · changes since r-1.26.0_

## Features
- Chat agents get a full-page editor at Admin → Chat Agents, covering the system
  prompt, model, tool allowlists, limits, and guest-chat appearance settings.
- Guest conversations are now stored per agent and browsable (with JSON export)
  under the agent's new Conversations tab — review exactly what visitors asked
  and how the agent answered, tool calls included.

## Fixes
- When an external API rejects a request, chat agents and the admin
  test-data-source tool now see the upstream status and the API's own error
  message instead of a generic "request could not be completed" — so an agent
  can correct a bad parameter instead of retrying blindly.
- Optional API query parameters left empty are now omitted from the request
  instead of being sent as empty values, which some APIs reject.

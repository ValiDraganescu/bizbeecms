# CMS v1.23.0

_2026-07-07 · changes since r-1.22.0_

## Features
- URL-based content locales: published pages now take their language from the URL prefix (e.g. `/fi/...`) instead of a cookie. The Language Switcher navigates to the prefixed URL, and `<html lang>` matches the URL's locale for correct SEO.
- Per-locale page slugs: each page can have its own slug per language, so localized URLs read naturally in each language. Internal links and the Language Switcher reverse-resolve to the right localized path, and sibling-slug collisions are guarded on save and in AI `create_page`.
- SEO: public `sitemap.xml`, plus canonical and `hreflang` alternate tags that account for localized slugs.
- Edge caching: opt a page in to Cloudflare edge caching with a per-page max-age. Publish / unpublish / delete and content-changing edits purge the cache automatically (per-page and shared-tag).
- Operator guide documenting URL locales, localized slugs, SEO, and the edge-cache opt-in / purge behavior.

## Fixes
- Published pages no longer carry the admin UI's locale/message data into their cached payloads (kills an Accept-Language cache-poisoning path and a large payload bloat).
- `/sitemap.xml` is no longer wrongly stamped with a page's Cache-Control from a top-level wildcard page; root dotted files (sitemap/robots/llms.txt) are excluded from the edge-cache gate.

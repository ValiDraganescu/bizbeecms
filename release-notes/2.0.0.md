# CMS v2.0.0

_2026-07-23 · changes since r-1.28.0_

## Features
- AI usage is now metered from the provider's actual charged cost (not token
  estimates) into monthly billable/raw counters — the analytics Cost column
  shows real billable dollars.
- Monthly AI quota: when a site's quota is spent, the assistant, guest chat,
  translation, and image tools politely refuse until next month (guest
  messages are shown in the visitor's language). Uploads are never blocked —
  auto-description simply skips.
- The assistant's credit chip now shows "used $X of $Y" in customer dollars
  from the site's own meter, updating instantly.
- Model choices (chat agents, assistant, image description/generation,
  translation) are now picked from operator-curated model aliases with
  friendly names; previously stored models keep working unchanged.
- Chat-agent analytics moved to its own page with navigation from the agent
  list, editor, and conversations pages.
- New machine endpoint reports this site's monthly AI usage to the Project
  Manager for fleet dashboards.

## Breaking
- The "bring your own OpenRouter key" settings page is removed. Sites always
  use the platform-provisioned key; any previously saved custom key is
  ignored.
- The site's monthly AI limit now means the customer-facing billable quota;
  the underlying provider key cap is managed automatically as a safety
  circuit breaker.

## Other
- Curated model config is fetched from the Project Manager and cached locally
  (15 min), so curation changes propagate without redeploys and a PM outage
  never breaks AI calls.
- Background usage writes are guaranteed to land via ctx.waitUntil; shared
  helper extracted.

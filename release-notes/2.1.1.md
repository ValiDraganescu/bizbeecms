# CMS v2.1.1

_2026-07-23 · changes since r-2.1.0_

## Fixes
- Security/staleness fix: admin and API responses are now explicitly marked uncacheable at the worker, so no edge or intermediary cache can ever store and re-serve them. Previously, zone-level cache settings could cache an admin API response and serve it stale — and to unauthenticated visitors — until its edge TTL expired (seen as the AI model list not updating after changes in the Project Manager). Public endpoints that deliberately opt into caching (published pages, the markdown page variants, sitemap/llms.txt) are unaffected.

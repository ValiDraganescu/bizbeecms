# CMS v0.9.0

_2026-06-24 · changes since cms-v0.8.0_

## Features
- **OpenRouter AI provider.** CMS chat, AI tools, and translation now run on OpenRouter (default model `openai/gpt-4o-mini`) instead of the Cloudflare-native model. Operators can paste their own OpenRouter key in CMS settings, which takes precedence over the deployed key; with no key set, it falls back to Cloudflare Workers AI.
- **Per-Site AI key, minted automatically.** PM can mint a dedicated OpenRouter key per Site on deploy (with an optional monthly USD spend limit) and inject it into the CMS Worker — no manual key handling. Deploy surfaces a non-blocking warning if minting or key-decryption fails, and proceeds on the fallback key.
- **Password reset (forgot password).** Both PM and CMS gain a "Forgot password?" flow — request a reset link, set a new password from an emailed token; resetting kills existing sessions. Reset requests are enumeration-safe (EN/FI/ET).
- **Sign out.** CMS admin gains a logout button (`POST /api/auth/logout`) in the admin footer (EN/FI/ET).
- **Operators show under their real email.** SSO now records the CMS user under their verified PM email and backfills any earlier placeholder row, so operators appear under their real address in the user list.

## Fixes
- Translation no longer 502s on Sites with an OpenRouter key — it uses the shared model catalog instead of a hardcoded Cloudflare model id.

## Security
- **BREAKING (operators): per-Site Google OAuth client required.** The shared, deployer-injected "Sign in with Google" client is gone. Each Site now configures its OWN Google OAuth client in CMS settings, stored encrypted in that Site's database. The Google login button only appears once a Site has its own client configured. Existing Sites relying on the old shared client must re-enter their Google credentials in CMS settings.
- Google `id_token` is now verified by RS256 JWK signature (hardening).
- Brute-force throttle on `/api/auth/login` and rate-limit on `/api/auth/forgot` (sliding-window, `429 + Retry-After`).
- Opportunistic pruning of spent password-reset rows, expired sessions, and aged-out login-attempt rows (no cron — swept inline).

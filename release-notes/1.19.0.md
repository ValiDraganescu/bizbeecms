# CMS v1.19.0

_2026-07-02 · changes since r-1.18.0_

## Features
- Dynamic / parameter-driven pages: a page path can carry parameters (e.g. `/:city-slug`, `/offers/:offer-slug`, nested wildcards under a static parent). The matched param resolves at render time — bind sections to it, look up the matching collection item, and surface it in the page's title/meta. Un-matched wildcards return a proper 404.
- Page params flow into bindings and the AI page tools, so the assistant can author param-driven pages and bind blocks to the resolved item.

## Fixes
- Deploy: a timed-out or killed deploy no longer wedges a Site in the "deploying" state forever — the state now clears so the Site can be redeployed.

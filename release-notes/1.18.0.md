# CMS v1.18.0

_2026-07-02 · changes since r-1.17.0_

## Features
- External data sources: connect pages to external HTTP APIs. Define reusable sources with saved requests (all methods, `{placeholder}` params), five auth modes (none / basic / bearer / header key / query key) plus OAuth2 client-credentials, per-request caching, and retries — all through one central fetch engine.
- Bind components and lists to either a Collection **or** an API source: pick a saved request, pass params from props, and map response fields with dot-paths (with sample-data-driven suggestions).
- Data Sources admin UI: create and manage sources and saved requests, with write-only encrypted secrets, per-request cache config, live test calls, and cache purging (per-request / per-source / global).
- AI assistant can drive data sources end to end — list/create/test a source, bind components and lists to API responses, and read an on-demand data-sources guide; plus inline data-sources context on the admin page.
- Built-in **Form** block: a native `<form>` whose child inputs are the fields, submitting to either an API saved request or an opt-in Collection (submissions land as drafts). Dual submit mode (native form-data + progressive-enhancement fetch/JSON), page-builder Form settings panel, per-collection "public submissions" toggle, and AI tools (`create_form` with an optional child component, `bind_form`).

## Fixes
- Inspector bind panel now edits API-keyed bindings (previously showed "data source — none —").
- Bind/List panel copy is source-agnostic now that API sources are selectable.
- "Data sources" sidebar nav item now shows its icon.

## Other
- Security hardening in the fetch engine: same-host-only redirect follow (SSRF/auth-leak safe), a 5MB streaming response size cap, and query-auth secrets preserved across same-host redirects.

# CMS v0.8.0

_2026-06-23 · changes since cms-v0.7.0_

## Features

### Per-site OpenRouter API key (new)
- Each site can now use its **own** OpenRouter API key for the AI assistant, set in ProjectManager → Site settings (write-only field: shows "key set" / "no key", never reveals the stored value, with a "Clear key" action).
- The key is **encrypted at rest** in ProjectManager (AES-256-GCM), decrypted only at deploy time, sent to the deployer over HTTPS, and installed as a real **secret** on that site's CMS Worker — never a plaintext variable, never written to a deploy log.
- Graceful fallback at every step: a site with no key (or a key that can't be decrypted) falls back to the deployer's global OpenRouter key, and if that's also unset, to the built-in Workers-AI provider. A missing key never breaks a deploy.

## Other
- New ProjectManager secret `SITE_SECRET_KEY` (the encryption key) and a `sites.openrouterApiKeyEncrypted` column + migration.
- The deployer now sets the CMS Worker's OpenRouter key via `wrangler secret put` instead of a build-time variable.

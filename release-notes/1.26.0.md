# CMS v1.26.0

_2026-07-21 · changes since r-1.25.4_

## Features
- **Guest chat agents.** A new *Chat Agents* admin page lets you configure chatbots that talk to logged-out visitors on your published pages. Each agent has its own system prompt, model, welcome message, usage limits, and an explicit allowlist of the data-source requests and collections it may touch.
- **GuestChat block.** Place a chat agent on any page from the Page Builder — as an inline panel or a floating bubble. The visitor never sees or chooses the agent's model, prompt, or tools: the server re-reads the block from the published page and loads its configuration.
- **Safe by default for guests.** A guest bot's queries only ever see published content, and anything it creates or updates lands as a draft for you to review. Per-IP and per-site rate limits (per minute, per day, per conversation) cap abuse and cost, and token usage is counted per day.
- **The admin assistant can manage agents.** Ask the AI assistant on the Chat Agents page to list, create, update, or delete an agent; it has a built-in playbook covering limits, allowlist shapes, and the safety model.
- **More file types in the media gallery.** PDF, plain text, Markdown, CSV, and JSON uploads are now accepted alongside images. Files whose type the browser doesn't report — `.md` most often — are recognised from their filename instead of being rejected.

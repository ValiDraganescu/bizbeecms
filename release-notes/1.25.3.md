# CMS v1.25.3

_2026-07-08 · changes since r-1.25.2_

## Fixes
- Publishing a component no longer leaves a phantom "unpublished changes" draft behind. Two things caused the notice to reappear the instant you published: pending edits to prop defaults were saved a moment too late (after the publish) and re-created the draft, and a formatting-only difference between how the Develop editor and the AI write the same markup was mistaken for a real change. Both are fixed — after you publish, the component is genuinely clean.

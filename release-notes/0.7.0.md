# CMS v0.7.0

_2026-06-22 · changes since cms-v0.6.0_

## Features

### Content collections (new)
- Define your own data collections — each becomes a real database table, created safely behind a hardened SQL fence (no raw SQL ever reaches the table layer).
- Full admin UI: create and edit a collection's schema with a field-type picker, then manage its items in a table with the right input per type (date/time, number, select, boolean, multiselect, rich text).
- Structured search, filter, sort and pagination over collection items.
- Bind collection data into pages: a single block can pull one record's fields into its props, or a new built-in **List** block can stamp a component once per matching record — all authored visually in the page builder, with graceful empty states.
- AI assistant can create collections, add/update/archive items, query data, and wire up component bindings — all through structured tools, never raw SQL.

### CMS-local accounts & roles (new)
- Sign in directly in the CMS with email + password, **Sign in with Google**, or PM single-sign-on — no more forced redirect to ProjectManager.
- Four-role hierarchy (SuperAdmin / Admin / Manager / Editor) with server-enforced permissions on every admin route.
- Invite teammates by email (sent via Cloudflare Email) with a role-limited invite form; accept-invite flow creates the account and signs them in.
- User-management screen: list users and pending invites, change roles, and remove users — all gated by who's allowed to do what.

### Manage a site from your own AI tools (new)
- Each deployed CMS exposes a remote MCP server at `/mcp`, so you can connect Claude Code (or any MCP client) and manage the site with your own model.
- Mint and revoke per-site API keys in **Settings → API Keys**, with a copy-paste "Connect Claude Code" snippet.

### Component kits
- Tag components, then export everything with a tag as a single installable kit bundle — and import a kit in one step.

### AI provider
- The AI assistant now runs on OpenRouter by default (with the built-in Workers-AI provider as fallback), and its model picker lists the full OpenRouter catalog.

## Other
- Hardened the runtime SQL fence with a parsing validator and an attack-string test corpus; added a schema-rebuild planner for future field drop/rename.
- Numerous database migrations for the new auth, roles, tags, API-key, invite and collection tables (applied automatically on deploy).
- Trilingual (EN/FI/ET) coverage for every new screen.

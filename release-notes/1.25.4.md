# CMS v1.25.4

_2026-07-21 · changes since r-1.25.3_

## Fixes
- "Sign in with BizbeeCMS" works again on deployed Sites. The login page built
  the SSO return URL with `http://` (the in-Worker proxy hop mis-reports the
  protocol), which the manager rejected with `badReturn`. The return URL is now
  always `https://`.

## Other
- OpenRouter is now the sole AI provider. The Workers AI fallback and the AI
  Gateway plumbing were removed — every Site deploys with an OpenRouter key, so
  the fallback was unreachable. If a Site somehow has no key, the AI assistant
  endpoints now respond 503 instead of silently switching providers.

# CMS v1.28.0

_2026-07-22 · changes since r-1.27.0_

## Features
- Guest chatbots now remember their conversation server-side: what the bot's tools already learned (looked-up records, policies) carries across messages instead of being re-fetched every turn, making replies faster, cheaper, and free of "let me look that up again" repetition.
- Chat requests are provider-cache friendly: the payload for a conversation only ever grows (stable system prompt, append-only transcript), so prompt caching keeps repeat-message costs down.
- Data-source tools on a chat agent can declare `requiredParams` — parameters the guest bot must always pass non-empty. An under-specified call (e.g. a search without a date window) is refused by the server with a corrective error, whatever the prompt says.
- Chat-agent welcome messages can be localized: provide a per-language greeting (locale object) and the widget greets each visitor in their own content locale, with fallback to the site default. The admin editor gains a per-locale input with the standard language tabs.

## Fixes
- Chat widget: when one assistant reply spans several tool rounds, the text after a tool call now starts on a new paragraph instead of running on inline after the previous sentence.
- Guest tool results are no longer over-truncated: the guest cap now matches the shared 15k sample bound, and the truncation notice tells the guest bot to narrow its filters instead of referencing an admin-only field it can't see.

# CMS v1.25.2

_2026-07-08 · changes since r-1.25.1_

## Fixes
- Component previews in the Develop workbench, the page-builder rail, and the AI assistant's screenshots now always reflect the latest draft. Previously, after you edited a component the preview could keep showing the old version because a deployed Site was caching preview pages in the browser for up to a year. Previews are no longer browser-cached (only the read-only components gallery still caches, and it busts on every save), and existing stale cache entries are bypassed automatically.

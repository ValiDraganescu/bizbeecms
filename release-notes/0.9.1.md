# CMS v0.9.1

_2026-06-24 · changes since cms-v0.9.0_

## Fixes
- **Deploy no longer fails at the `secret` step.** The OpenRouter API key is now set purely as a Worker secret. It was also declared as a plain-text var, so the post-deploy `wrangler secret put` collided on the binding name (`10053: Binding name 'OPENROUTER_API_KEY' already in use`) and failed every deploy. Removing the var declaration lets the secret set cleanly; OpenRouter chat/translate is unaffected at runtime.

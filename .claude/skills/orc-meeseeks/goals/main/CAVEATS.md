# Caveats — main
Read every line before working. Each entry was learned the hard way by a previous Meeseeks.

- **Cloudflare Workers caps PBKDF2 at 100_000 iterations.** `crypto.subtle.deriveBits` with `iterations > 100000` throws `NotSupportedError: Pbkdf2 failed: iteration counts above 100000 are not supported` AT RUNTIME ON THE DEPLOYED WORKER ONLY (builds + tsc pass; local node has no such cap). This silently broke ALL registration on the live deploy — `hashPassword` threw before any DB write, so no user was ever created and login was impossible, surfacing only as a generic 500 "server error". `src/lib/auth/password.ts` `ITERATIONS` is pinned to `100_000` (the platform ceiling, under the OWASP 210k floor). The stored hash is self-describing (`pbkdf2$<iter>$...`) so this can be raised if the limit ever lifts. DO NOT raise it above 100000.
- **Diagnosing live Worker 500s:** `wrangler tail` does NOT surface exceptions that OpenNext catches into a 500 response (outcome stays "ok", `exceptions[]` empty). To get the real error, wrap the route handler in try/catch and return the message in a temporary `_debug` JSON field, deploy, curl it, read `_debug`, then remove. This is how the PBKDF2 cap above was found.
